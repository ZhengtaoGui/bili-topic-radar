# Web 看板 v2:深色科技风 + 表单对齐修复(给 Codex 的美术方向)

目标:把看板升级成**深色、有科技感的"雷达控制台"**风格,并修掉表单对齐问题。
**保留 v1 所有人话文案、tooltip、教程内容、徽章规则、`?demo=` 链接**(见 web_redesign_spec.md);
这次只动视觉与布局,不丢任何信息、不改后端路由/数据契约、不引新依赖、test_web.py 继续通过。

Claude 定方向,Codex 放手发挥设计天赋——下面是约束和品味基线,细节你来。

## 一、主题:雷达控制台(Radar Console)

整体气质:深空、冷静、精密。像一个"选题雷达"的操作台,而不是普通后台表格。

## 二、配色(深色)

- 背景:深空底 `#0a0e14` ~ `#0d1117`,可叠一层很淡的径向渐变或点阵网格纹理;
  hero 角落可放一个**极淡的雷达扫描光晕/同心圆**做主题暗示(别太抢眼)。
- 面板:玻璃拟态——`rgba(255,255,255,0.04)` + `backdrop-filter: blur(12px)`,
  1px 边框 `rgba(255,255,255,0.08)`,可加极淡内发光。
- 主强调色:青/teal 霓虹 `#2dd4bf` / `#22d3ee`;次강调:电光紫 `#818cf8`。
  主按钮 hover 带轻微辉光(box-shadow glow)。
- 文字:主 `#e6edf3`,次 `#8b97a7`,边线 `rgba(255,255,255,0.08)`。
- 链接:青色 `#22d3ee`。

## 三、关键组件质感

- **Hero**:标题可用青→紫渐变文字或淡辉光;旁边/背后放一个雷达 scope 暗纹
  (同心圆 + 一道扫描线)强化"雷达"身份。副标题、免责小字保留。
- **统计卡**:大号数字(可用等宽数字字体 `ui-monospace`/`tabular-nums`),
  数字带极淡辉光;解读徽章用霓虹感半透明 pill:
  绿 `#34d399`(利好/上升/好进)、琥珀 `#fbbf24`(中性)、红 `#fb7185`(不利/降温/锁死),
  各带对应淡色背景 + 轻 glow。卡片 hover 微微抬升/边框提亮。
- **表格**:深色行、hover 行轻微高亮(青色淡背景);表头粘性、半透明深色;
  命中搜索词标签 = 深色半透明 pill;高赞评论(≥100)高亮保留(用青/金强调)。
- **输入框**:深底 `#0f141c`,聚焦时边框/外发光变青色。
- **教程弹窗(dialog)**:同款深色玻璃面板,别变回亮色。

## 四、表单对齐修复(重点!现在很丑)

现状问题:左边"选题关键词"是矮输入框、右边"额外搜索词"是高文本框,用 `align-items:end`
硬底对齐,导致高低参差;"自动加常见后缀"和"开始扫描"位置也别扭。

改成**顶部对齐、分区清晰**的布局:
- 推荐两栏:
  - 左栏(约 40%):上=**选题关键词**输入框,下=**抓多少热评**输入框(纵向堆叠)。
  - 右栏(约 60%):**额外搜索词**标签 + 文本框,文本框**下方右侧**放「+ 自动加常见后缀」小按钮。
  - 两栏**顶部对齐**(`align-items: start`),不要再用底对齐。
- **「🚀 开始扫描」单独占一整行**(在两栏下方),全宽或左对齐的大按钮 + 右侧那行说明小字。
- 各 label 统一"标签在上、帮助文字在下、控件再下"的三段式,行高一致。
- 移动端自动单列堆叠。
- 整体留白均匀,控件不要忽高忽低。
